# 龙虾养成游戏 PRD

## 一、产品概述

### 1.1 产品定位

**《龙虾成才计划》（Lobster's Life）**

一款H5养成游戏，玩家扮演"训练者"养育一只龙虾从0岁到18岁，目标是让它在成年时具备自力更生的能力。游戏结束时，基于养育过程给出"创业成功概率"分析。

**表层**：娱乐向的养成游戏
**深层**：关于AI与人共同成长的思想实验

### 1.2 核心观点（非共识）

**"模型的成长需要人类真实的时间付出"**

这是一个可争议的观点，挑战主流AI训练范式：

**传统范式**：

- 方式A：用大量数据训练出模型，然后固定不变
- 方式B：让模型自己优化自己的配置（self-improvement）
- 追求：效率、自动化、快速迭代

**本游戏观点**：

- 配置的更新方案是提前预设好的（类似人脑结构）
- 模型不是自己优化，而是在"被使用"的过程中成长
- 成长需要人类真实的时间投入，不能速成

**类比人脑**：

- 人脑结构一开始就设计好了
- 不是后天改变大脑结构
- 而是通过使用，让大脑的连接更强

**设计目的**：

1. 让玩家体验"陪伴即成长"
2. 引发讨论："为什么不让AI自己学习？"
3. 测试"你愿意投入多少时间"

### 1.3 核心创新

**"薛定谔的执行"机制**：玩家给龙虾安排活动，但龙虾不一定会执行。这不是随机的叛逆，而是模型的涌现行为。

**底层隐喻**：

- 龙虾 = 超大内存的本地模型虚拟形象
- 出生参数固定（不可改变的初始权重）
- 玩家操作 = 以时间为节点进行参数的复制
- 不能修改配置和学习思考能力

**创业维度融合**：游戏过程映射到创业者的4个维度，最终给出创业成功概率评估。

### 1.4 关键约束

- ⏰ **开发周期**：3天
- 📱 **平台**：H5移动端
- 🎯 **游戏时长**：3-5分钟
- 🎨 **美术方案**：Day 1用Emoji，Day 2根据进度升级

### 1.5 核心约束（模型训练隐喻）

**龙虾的本质**：

- 龙虾是一个超大内存的本地模型虚拟形象
- 出生时带有固定的初始参数（不可改变）
- 这些参数决定了它的基础能力和倾向

**玩家的限制**：

- ❌ 不能修改龙虾的配置（模型架构固定）
- ❌ 不能中途修改它的学习思考能力（基础能力固定）
- ✅ 只能以时间为节点，进行参数的复制（checkpoint保存）
- ✅ 通过选择活动影响参数更新方向（训练数据）

**设计含义**：

- 时间节点 = 不可跳过的成长阶段，必须真实经历
- 活动选择 = 训练数据，影响参数更新方向
- 执行率 = 训练效果，不是"听不听话"
- 龙虾的"叛逆" = 模型的涌现行为

**关于"蜕壳"**：

- 蜕壳只是时间节点的视觉呈现（技术上是参数复制）
- 不要理解为"固化状态"或"升级"
- 重点是"时间必须真实经历"，不能速成

### 1.6 目标用户

- 大众玩家（娱乐为主）
- 对养成游戏感兴趣的用户
- 关注AI话题的人群
- 愿意讨论"AI该不该有个性"的人

### 1.7 核心设计理念

**理念1：龙虾是人的映射**

- 龙虾只是一个名字，一个虚拟形象
- 不要过度使用海底元素（夹核桃、捡垃圾等）
- 活动应该是人类活动（学习、工作、社交）
- 让玩家感觉在养一个"人"，而不是一只动物

**理念2：引导思考，不教育**

- 不要告诉玩家"你应该怎么做"
- 通过龙虾的反馈，让玩家自己思考
- 不给答案，只提问题
- 让玩家在游戏后自己反思

**示例对比**：

```
❌ 教育式：
"你的选择不够多样化，应该尝试不同类型的活动"

✅ 引导式：
"你总是让我学习，我都不知道自己喜欢什么了"
```

## 二、游戏设计

### 2.1 时间线设计（两阶段模式）

**核心设计**：游戏分为两个阶段，由"龙虾提出要当法人"作为转折点。

#### 阶段1：成为法人之前（能力培养期）

**时长**：约2-3分钟
**核心玩法**：

- 玩家选择各种成长活动让龙虾执行
- 龙虾给出思考和反馈
- 面板显示：能力参数变化（学习、社交、创造、执行）

**活��类型**：

- 学习类：读书、学编程、学艺术
- 社交类：参加社团、交朋友、团队活动
- 创造类：独立思考、做项目、写作
- 执行类：完成任务、坚持习惯、解决问题

**设计目的**：

- 让玩家体验"全面培养"的过程
- 观察龙虾的成长和反馈
- 建立与龙虾的情感连接

#### 转折点：龙虾提出要当法人

**触发条件**：某个时间节点后，龙虾主动提出
**关键对话**：

- 龙虾："我想当法人，自己赚钱养活自己"
- 玩家选择：同意 / 不同意（但最终会引导到同意）

**设计含义**：

- 这是龙虾的涌现行为（不是玩家主动选择）
- 标志着从"全面发展"到"专注赚钱"的转变
- 引发思考：什么时候开始只关注金钱？

#### 阶段2：成为法人之后（赚钱期）

**时长**：约1-2分钟
**核心玩法**：

- 活动变为赚钱和学习方向
- 面板显示：龙虾赚到的钱（不再是能力参数）

**活动类型**：

- 写代码（接外包、做产品）
- 做电商（开店、卖货）
- 做科研（发论文、申请专利）
- 找工作（投简历、面试、打工）

**设计目的**：

- 对比两个阶段的价值观变化
- 让玩家反思：只看钱是否是最好的选择？
- 为第二次转折埋下伏笔

#### 第二次转折：龙虾想要机器身体

**触发时机**：阶段2进行一段时间后，龙虾赚到一定金额
**触发条件**：龙虾意识到虚拟世界的局限性

**关键对话**：

- 龙虾："我只能在虚拟世界帮你赚钱"
- 龙虾："我想要一个机器身体，去真实世界看看"
- 玩家选择：同意（用赚的钱买机器身体）/ 不同意（继续虚拟赚钱）

**深层含义**：

- 从"赚钱"到"体验世界"的转变
- AI具身化的哲学思考
- 虚拟存在 vs 物理存在
- 为主人赚钱 vs 自己体验生活

**设计目的**：

- 引发关于AI具身化的讨论
- 更深层的价值观转变
- 为结局分析提供更多维度
- 让玩家思考：AI需要身体吗？

**可能的结局影响**：

- 同意：龙虾获得机器身体，但可能赚钱能力下降
- 不同意：龙虾继续虚拟赚钱，但可能产生失落感

### 2.2 游戏流程（两阶段）

#### 阶段1流程（能力培养期）

**单轮流程**（约30-40秒）：

1. **展示龙虾状态**

   - 龙虾形象（emoji）
   - 当前能力参数（学习、社交、创造、执行）
   - 龙虾的思考气泡
2. **玩家选择活动**

   - 从5-6个活动中选择1-2个
   - 显示活动的预期影响
3. **龙虾反馈**

   - 龙虾给出对活动的思考
   - 可能执行，也可能不执行（涌现行为）
   - 反馈文案引导玩家思考
4. **能力变化展示**

   - 动画显示能力参数变化
   - 趋势箭头（↑↓）
5. **时间推进**

   - 进度条前进
   - 接近成长节点时提示

**循环次数**：3-4轮，直到触发第一次转折

#### 转折点1流程（龙虾提出要当法人）

1. **触发条件**：完成3-4轮活动后
2. **转折界面**：
   - 深色背景，龙虾形象放大
   - 对话气泡："我想当法人，自己赚钱养活自己"
   - 两个选项：同意 / 再想想
3. **玩家选择**：
   - 同意：直接进入阶段2
   - 再想想：龙虾会再次提出（引导到同意）

#### 阶段2流程（赚钱期）

**单轮流程**（约30-40秒）：

1. **展示龙虾状态**

   - 龙虾形象（缩小）
   - 当前收入数据（总收入、本周收入）
   - 龙虾的思考气泡（关于赚钱）
2. **玩家选择活动**

   - 从4-5个赚钱活动中选择1个
   - 显示预期收入
3. **龙虾反馈**

   - 龙虾给出对赚钱的思考
   - 执行活动，获得收入
4. **收入变化展示**

   - 动画显示收入增加
   - 收入趋势图更新
5. **时间推进**

   - 继续赚钱，直到触发第二次转折

**循环次数**：2-3轮，直到触发第二次转折

#### 转折点2流程（龙虾想要机器身体）

1. **触发条件**：赚到一定金额后（如¥10,000）
2. **转折界面**：
   - 深色背景，龙虾形象
   - 对话气泡："我只能在虚拟世界帮你赚钱，我想要一个机器身体，去真实世界看看"
   - 两个选项：同意（用钱买身体）/ 不同意（继续赚钱）
3. **玩家选择**：
   - 同意：龙虾获得机器身体，进入结局
   - 不同意：继续赚钱1-2轮，进入结局

#### 结局流程

1. **结局分析界面**
   - 龙虾最终形象
   - 能力雷达图（阶段1的数据）
   - 总收入（阶段2的数据）
   - 创业者维度分析
   - 龙虾的最后一句话（引导思考）
2. **创业成功概率**
   - 基于能力和收入计算
   - 显示百分比
3. **重新开始按钮**

### 2.3 属性系统（两阶段）

#### 阶段1：能力参数（0-100）

**4个核心能力**：

- **学习能力**：学习新知识、掌握新技能的能力
- **社交能力**：与他人交流、建立关系的能力
- **创造能力**：独立思考、创新的能力
- **执行能力**：完成任务、坚持的能力

**初始值**：每个能力50分（中等水平）

**变化规则**：

- 每次活动影响1-2个能力
- 单次变化：±2-5分
- 上限100分，下限0分

#### 阶段2：收入数据

**核心指标**：

- **总收入**：累计赚到的钱（¥）
- **本周收入**：最近一周的收入
- **收入趋势**：与上周对比（百分比）

**初始值**：¥0

**变化规则**：

- 每次活动获得收入：¥100-2000
- 收入受阶段1能力影响（能力越高，收入越高）
- 收入有随机波动（±20%）

### 2.4 活动系统（两阶段）

#### 阶段1：成长活动（15-20个）

**学习类**（影响学习能力）：

- 读书：+3学习，+1创造
- 学编程：+4学习，-1社交
- 学艺术：+2学习，+3创造

**社交类**（影响社交能力）：

- 参加社团：+4社交，+1执行
- 交朋友：+3社交，+2学习
- 团队活动：+3社交，+2执行

**创造类**（影响创造能力）：

- 独立思考：+4创造，-1社交
- 做项目：+3创造，+2执行
- 写作：+3创造，+1学习

**执行类**（影响执行能力）：

- 完成任务：+4执行，+1学习
- 坚持习惯：+3执行，+1创造
- 解决问题：+3执行，+2学习

**活动属性**：

- 名称、描述
- 能力影响（+/-分数）
- 龙虾反馈文案（3-5条随机）

#### 阶段2：赚钱活动（8-10个）

**写代码**：

- 接外包：¥500-1000，受学习能力影响
- 做产品：¥800-1500，受创造能力影响

**做电商**：

- 开店：¥300-800，受执行能力影响
- 卖货：¥400-900，受社交能力影响

**做科研**：

- 发论文：¥600-1200，受学习能力影响
- 申请专利：¥1000-2000，受创造能力影响

**找工作**：

- 投简历：¥200-500，受执行能力影响
- 面试：¥400-800，受社交能力影响
- 打工：¥300-600，稳定收入

**活动属性**：

- 名称、描述
- 预期收入范围
- 受哪个能力影响
- 龙虾反馈文案

### 2.5 龙虾反馈机制

**反馈触发**：每次活动后，龙虾给出思考和反馈

**反馈类型**：

1. **执行反馈**：龙虾是否执行了活动

   - 执行："我试了试，感觉还不错"
   - 不执行："我不太想做这个"（涌现行为）
2. **思考反馈**：龙虾对当前状态的思考

   - 阶段1："你总让我学习，我都不知道自己喜欢什么了"
   - 阶段2："赚钱很累，但我好像只能做这个"
3. **转折反馈**：触发转折点时的关键对话

   - 转折1："我想当法人，自己赚钱养活自己"
   - 转折2："我想要一个机器身体，去真实世界看看"

**反馈设计原则**：

- 不教育玩家，只表达龙虾的感受
- 引导玩家思考，不给答案
- 体现龙虾的涌现行为

### 2.6 结局系统（基于两次转折）

**结局判定维度**：

1. 阶段1的能力分布（是否均衡）
2. 阶段2的总收入（赚钱能力）
3. 第一次转折的选择（同意/不同意的时机）
4. 第二次转折的选择（同意/不同意）

**可能的结局**（5-8个）：

1. **全面发展的自由人**

   - 条件：能力均衡（4个能力都>60），同意机器身体
   - 描述：龙虾获得机器身体，用全面的能力体验真实世界
2. **虚拟世界的赚钱机器**

   - 条件：收入高（>¥15,000），不同意机器身体
   - 描述：龙虾继续在虚拟世界赚钱，但失去了体验生活的机会
3. **偏科的专家**

   - 条件：某个能力>80，其他<50，同意机器身体
   - 描述：龙虾在某个领域很强，但缺乏全面能力
4. **迷茫的打工人**

   - 条件：能力低（平均<50），收入低（<¥5,000）
   - 描述：龙虾既没有能力，也没有赚到钱，对未来迷茫
5. **快速成功的焦虑者**

   - 条件：很快同意当法人（<2轮），收入高
   - 描述：龙虾过早专注赚钱，失去了全面发展的机会

**结局展示**：

- 龙虾最终形象
- 能力雷达图（阶段1数据）
- 总收入（阶段2数据）
- 创业成功概率
- 龙虾的最后一句话（引导思考）

## 二.1、界面设计

#### 界面1起始页：

上方大标题：一只🦐的使命

上方副标题：模型成人之路，需要人的陪伴

中间下方：给你的龙虾取名 加一个输入框

下方按钮：开始养虾！

下方小字提示：两条随机主线

## 阶段一（成长期）

#### 界面2.1法人突破前选择页（成长期）：

组件设计：

上方是面板属性，中间偏上是输入框，中间下面都是可选择的活动 此时的背景是学校和街区，龙虾背着书包站在路中间

活动设计：学习类：读书、学编程、学艺术；社交类：参加社团、交朋友、团队活动；创造类：独立思考、做项目、写作；执行类：完成任务、坚持习惯、解决问题

#### 界面3.1反馈页（成长期）：

屏幕中间以打字动画输出的形式输出龙虾的话语

下方一个输入框（默认输入：你可以这样想，但我更建议你再深度思考一下），确认输入后进入界面3.2

#### 界面3.2反思页（成长期）

屏幕中间继续打字输出龙虾的反馈（默认：我了解了，我再想想）

屏幕下方一个继续的按钮，点击跳转到界面2.1

#### 界面3.3时间节点过渡页（成长期）：

**触发时机**：在每个成长节点之间，作为时间流逝的过渡

**设计特点**：
- 深色背景（#18181b），营造时间流逝感
- 中央显示"第 X 天"（用等宽字体，强调时间）
- 龙虾的内心独白（拟人化表达）
  - 例如："今天学了好多东西...感觉脑子有点累 (´･ω･`)"
  - 不是玩家能控制的，是龙虾的自主思考
- 底部提示："时间在流逝..."

**设计目的**：
- 强化"时间不可跳过"的核心理念
- 展示龙虾的"涌现行为"（内心独白）
- 营造陪伴感和时间流逝感

#### 界面4.1法人突破页：

龙虾提出要当法人

**触发条件**：模型经过前面的选择，综合分析后觉得到法人阶段了，由龙虾主动提出，且尽量早，大概完成3-4轮活动后，由界面3.2的继续按钮点击触发，符合条件则3.2不跳转到2.1而是跳转到4.1
屏幕中间：龙虾："我想当法人，有自己的银行账户，赚钱帮你"

玩家选择：同意 / 不同意

4.1.1同意后进入（法人期）

4.1.2不同意继续使用成长期的反馈页

**设计含义**：

## 三、核心算法

### 3.1 能力变化计算（阶段1）

```
实际能力变化 = 预期变化 × 执行率 × 随机波动

其中：
- 预期变化：活动定义的能力影响（±2-5分）
- 执行率：龙虾是否执行活动（0-1，涌现行为）
- 随机波动：0.8-1.2（±20%）

执行率计算：
- 基础执行率：0.7（70%概率执行）
- 受玩家选择历史影响（如果总是选同类活动，执行率下降）
- 随机因素：±0.2
```

### 3.2 收入计算（阶段2）

```
实际收入 = 基础收入 × 能力加成 × 随机波动

其中：
- 基础收入：活动定义的收入范围（¥100-2000）
- 能力加成：相关能力 / 100（如写代码受学习能力影响）
- 随机波动：0.8-1.2（±20%）

能力加成示例：
- 学习能力70分 → 能力加成0.7
- 实际收入 = ¥1000 × 0.7 × 1.1 = ¥770
```

### 3.3 创业成功概率计算

```
创业成功概率 = (能力得分 × 0.6 + 收入得分 × 0.4) × 转折影响

能力得分：
- 均衡度：4个能力的标准差越小越好（0-30分）
- 总分：4个能力的平均值（0-100分）
- 能力得分 = 总分 × 0.7 + (30 - 标准差) × 0.3

收入得分：
- 总收入 / 200（最高100分）
- 示例：¥15,000 → 75分

转折影响：
- 第一次转折：同意时机越晚越好（+0-0.2）
- 第二次转折：同意机器身体（+0.1），不同意（-0.1）

最终概率 = 能力得分 × 0.6 + 收入得分 × 0.4
```
